Architectural concern: GameEngine has grown to handle UI, game loop, win/lose, and restart — consider extraction (#59).
- **2026-03-10 — ARCHITECTURE.md refreshed to match current state.** Updated documentation to reflect all 14 commands (TalkCommand, SaveCommand, LoadCommand, ExamineCommand added since v1), Engine systems (NarratorEngine, GameMessages, ColorConsole, GameStateExtensions, JsonWorldLoader), NPC/Dialogue/NarratorVariant models, drone threat system (DroneThreatLevel, DroneThreatThreshold, HighRiskRoomIds), restart-on-death via stateFactory (Issue #29), save/load format with threat/locks (Issue #35), dynamic descriptions via NarratorEngine (Issue #33), TalkCommand setting {npc_id}_met flags (Issue #46), 227+ test coverage. Removed stale content (old world map, outdated command signatures). Committed on current branch (squad/46-viktor-met-flag). Document now serves as accurate reference for current architecture.
- **2026-03-10 — DroneThreatSystem extracted (Issue #44).** Created `src/MyGame/Engine/DroneThreatSystem.cs`. The class holds a `GameState` reference and exposes three methods: `IsHighRiskRoom()` (checks current room against `HighRiskRoomIds`), `Increment()` (increments `DroneThreatLevel`, returns warning string or null, sets `HasLost`/`IsRunning` at threshold), and `Reset()` (sets level to 0). `DroneThreatLevel` and `DroneThreatThreshold` remain on `GameState` for save/load compatibility — `DroneThreatSystem` reads and writes them through the state reference. `GameEngine` holds a `_droneSystem` field, instantiated from the constructor state; on try-again restart the field is re-created alongside `_state`. The 15-line inline block in `GameEngine` is replaced by a 4-line delegation. Unused `prevRoomId` local removed. All 227 tests pass.
