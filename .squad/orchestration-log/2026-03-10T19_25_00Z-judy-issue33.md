## Summary

Created `GameStateExtensions.cs` with three extension methods on `GameState` for item lookup, eliminating duplicated search logic from four command classes.

## Work Completed

- **Created** `src/MyGame/Engine/GameStateExtensions.cs` with three public extension methods:
  - `FindItem(string noun)` — searches room then inventory
  - `FindRoomItem(string noun)` — searches room only
  - `FindInventoryItem(string noun)` — searches inventory only

- **Shared private predicate**: All three methods use identical match logic (exact ID match case-insensitive OR partial name contains case-insensitive)

- **Refactored four commands** to use extension methods:
  - `TakeCommand` — now uses `FindRoomItem`
  - `DropCommand` — now uses `FindInventoryItem`
  - `ExamineCommand` — now uses `FindItem`, removed private static helper
  - `UseCommand` — now uses `FindInventoryItem`

## Design Rationale

- **Single source of truth**: Predicate logic centralized; any future match semantics changes require only one edit
- **Extension method pattern**: `state.FindItem(noun)` reads naturally at call sites
- **Three scopes preserve intent**: Separate methods make search scope explicit (room-only, inventory-only, both) rather than hiding it in flags
